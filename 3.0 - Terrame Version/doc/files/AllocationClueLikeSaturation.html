AllocationClueLikeSaturation

Modification of the AllocationClueLike component. In this case the speed of change in each cell use a spatio-temporal variable, dynamically updated every year, that indicates if the cell is in a more consolidated or in a frontier area. The saturation threshold considers a 10x10 neighbourhood, discounting protected areas.

Arguments

  • allocationData: A table with two allocation parameters for each land use.
  • allocationData.changeLimiarValue: Threshold (or limier) refers to a given amount of the land use in each cell. After this limier, the speed of change of a given land use in the cell is modified.
  • allocationData.maxChange: Maximum change in a given land use allowed in a cell in a time step until (saturation) threshold.
  • allocationData.maxChangeAboveLimiar: Maximum change in a given land use allowed in a cell in a time step after (saturation) threshold.
  • allocationData.maxValue: Maximum value allowed for the percentage of a given land use in a cell (as a result of new changes - the original value can be out of the limits).
  • allocationData.minChange: Minimum change in a given land use in a cell in a time step until (saturation) threshold.
  • allocationData.minValue: Minimum value allowed for the percentage of a given land use in a cell (as a result of new changes - the original value can be out of the limits).
  • allocationData.static: Indicates if the variable can increase or decrease in each cell, or only change in the direction of the demand.
  • attrProtection: Database attribute indicating the percentage of protected areas to be excluded from the saturation level computation.
  • compareAllocationToDemand: Compares the demand to the amount of allocated land use/cover, then adapts elasticity.
  • complementarLU: The land use which will be recomputed in the end to sum exactly 100%.
  • computeChange: Compute the Allocation change based on the potential of the cell.
  • correctCellChange: Corrects total land use/cover types to 100 percent.
  • countAllocatedLandUseArea: Calculates total area allocated by the regression equations for each land use/cover type.
  • initElasticity: Handles with the elasticity initialize considering a single elasticity for each land use (all cells).
  • initialElasticity: Initial elasticity value (iterationFactor).
  • maxDifference: Maximum allocation error allowed for each land use in area.
  • maxElasticity: Maximum elasticity value which controls the allocation interaction factor.
  • maxIteration: Maximum number of iterations at each time step of the model.
  • minElasticity: Minimum elasticity value which controls the allocation interaction factor.
  • printAllocatedArea: Calculates and prints the allocated by the regression equations for each land use/cover type.
  • run: Handles with the rules of the component execution.
  • saturationIndicator: Name of a attribute which will be dynamically updated (and can be saved for calibration purposes).
  • updateAllocationParameters: Handles with the allocation parameters update.
  • verify: Handles with the verify method of a AllocationClueLikeSaturation component.

Usage

A1 = AllocationClueLikeSaturation
{
 maxDifference = 1643,
 maxIteration = 1000,
 initialElasticity = 0.1,
 minElasticity = 0.001,
 maxElasticity = 1.5,
 complementarLU = "floresta",
 saturationIndicator = "saturationLimiar",
 attrProtection = "uc_pi",
 allocationData =
 {
   {static = -1, minValue = 0, maxValue = 1, minChange = 0, maxChange = 1, changeLimiarValue = 1, maxChangeAboveLimiar = 0}, -- floresta
   {static = -1, minValue = 0, maxValue = 1, minChange = 0, maxChange = 1, changeLimiarValue = 1, maxChangeAboveLimiar = 0}, -- desmatamento
   {static = 1, minValue = 0, maxValue = 1, minChange = 0, maxChange = 1, changeLimiarValue = 1, maxChangeAboveLimiar = 0}   -- outros
 }
}
