PreComputedValuesINPE

A component (demand algorithm), where the demand inputs set by the user on the configuration file is used to distribute on the cells the correct among of this demand according to the parameters rules. The modeller informs pre-computed demand for each land use in each simulation year (see examples). These values may have been generated by any appropriate approach/method/external model.

Arguments

  • changeLuDirection: Invert the demand direction for an specific luIndex.
  • directionForInterpolation: Set the direction of the demand for each landUseTypesForInterpolation (optional).
  • finalYearForInterpolation: The year to be consider as the final year to calculate the demand interpolation.
  • getCurrentDemand: Return the current demand of the specified component.
  • getCurrentLuDemand: Return the current demand for an specific luIndex.
  • getCurrentLuDirection: Return the current demand direction for an specific luIndex.
  • getPreviousDemand: Return the previous demand of the specified component.
  • getPreviousLuDemand: Return the previous demand for an specific luIndex.
  • landUseTypesForInterpolation: The land use names within the data of the final year to calculate the demand interpolation.
  • run: Handles with the execution method of a PreComputedValuesINPE component.
  • verify: Handles with the verify method of a PreComputedValuesINPE component.

Return value:

    the modified component.

Usage

D1 = PreComputedValuesINPE
{
 annualDemand =
 {
   -- "floresta", "desmatamento", "outros"
   {7771, 2829, 12},   -- 2008
   {7766, 2834, 12},   -- 2009
   {7762, 2838, 12},   -- 2010
   {7757, 2843, 12},   -- 2011
   {7754, 2846, 12},   -- 2012
   {7751, 2849, 12},   -- 2013
   {7748, 2852, 12}  -- 2014
 }
}
