Obrigatórios		 Argumentos
Opicionais
"area" Porcentagem de área com alguma sobreposição com uma camada de polígonos. Os valores criados vão de zero (sem interseção) a um (área totalmente coberta por polígonos). Este algoritmo supõe que não há área de interseção entre cada par de polígonos da camada de referência. Soma as áreas de interseção do objeto com todos os polígonos da camada de referência. Por causa disso, se houver alguma sobreposição entre os polígonos da camada de referência, ela poderá criar valores de atributo maiores que um. attribute, layer split
"average" Média de valores quantitativos dos objetos que possuem alguma intersecção com a célula, sem levar em conta suas propriedades geométricas. Ao usar a área do argumento, calcula a média ponderada pelas proporções das respectivas áreas de interseção. Útil para distribuir atributos que representam médias, como a renda per capita. attribute, layer, select area, band, dummy, missing, pixel, split
"count" Número de objetos que possuem alguma sobreposição com a célula. attribute, layer dummy, pixel, split
"coverage" Porcentagem de cada valor qualitativo cobrindo a célula, usando polígonos ou dados rasterizados. Ele cria um novo atributo para cada valor disponível, sob a forma attribute.."_"..value, em que attribute é o valor passado como argumento para preencher e value representa os diferentes valores nos dados de entrada. A soma dos valores de atributo criados para uma determinada célula varia de zero a um, de acordo com a área da célula coberta por pixels. Note que esta operação não usa valor fictício, portanto, um atributo também será criado para os valores dummy. Ao usar shapefiles, tenha em mente o limite total de dez caracteres, pois ele remove os caracteres após o décimo no nome. Essa função será interrompida com um erro se dois nomes de atributos na saída forem iguais. attribute, layer, select band, missing, pixel, split
"distance" Distância para o objeto mais próximo. A distância é calculada a partir do centroide da célula até o ponto, linha ou borda mais próxima de um polígono. attribute, layer split
"maximum" Valor quantitativo máximo entre os objetos que possuem alguma intersecção com a célula, sem levar em conta suas propriedades geométricas. attribute, layer, select band, dummy, missing, pixel, split
"minimum" Valor quantitativo mínimo entre os objetos que possuem alguma intersecção com a célula, sem levar em conta suas propriedades geométricas. attribute, layer, select band, dummy, missing, pixel, split
"mode" Valor qualitativo mais comum dos objetos que possuem alguma interseção com a célula, sem levar em conta suas propriedades geométricas. Esta operação cria um atributo com valores de string. Sempre que houver dois ou mais valores com a mesma contagem, o valor resultante conterá todos eles separados por vírgula. Ao usar a área de argumento, ele usa o valor do objeto que possui uma cobertura maior. attribute, layer, select band, dummy, missing, pixel, split
"presence" Valor booleano que indica se algum objeto tem uma sobreposição com a célula. attribute, layer split
"stdev" Desvio padrão de valores quantitativos de objetos que possuem alguma intersecção com a célula, sem levar em conta suas propriedades geométricas. attribute, layer, select band, dummy, missing, pixel, split
"sum" Soma de valores quantitativos de objetos que possuem alguma intersecção com a célula, sem levar em conta suas propriedades geométricas. Ao usar a área de argumento, ele calcula a soma com base nas proporções da área de interseção. Útil para preservar a soma total em ambas as camadas, como o tamanho da população. attribute, layer, select area, band, dummy, missing, pixel, split

pixel : Um valor string que indica quando um pixel está dentro de um polígono:

Pixel Descrição
"centroid" (default) Um pixel está dentro de um polígono se o seu centróide estiver dentro do polígono. Recomenda-se usar esta estratégia quando se deseja manter a soma da quantidade de pixels na saída criada, e quando a resolução dos polígonos é maior que a resolução dos pixels. Se a resolução de polígonos for menor que a resolução de pixels, podem existir polígonos que não contenham nenhum pixel. Para representações celulares, quando as células foram criadas usando a varredura e a resolução de polígonos é consideravelmente menor que a resolução de pixels, um pixel pode pertencer a dois polígonos, já que seu centróide pode estar localizado no limite de ambos os polígonos.
"overlap" Um pixel é considerado dentro de um polígono se tiver alguma sobreposição. Dessa forma, um pixel pode pertencer a dois ou mais polígonos ao mesmo tempo. Essa estratégia é geralmente recomendada quando a resolução dos polígonos é menor que a resolução dos pixels. Essa estratégia leva mais tempo para ser executada.
