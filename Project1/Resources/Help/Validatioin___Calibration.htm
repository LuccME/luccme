To perform a validation and calibration access the Validation tab:

 

                                                                     Figure 1: Validation Window

 

In this version, two methodologies are available:

1.       Considering all the cellular space (Ex)

2.       Considering only the modified areas (Diff)

To perform a validation the model files must been generated (Make Files) and executed (Run Model).

If the files were correctly generated the Validate button will be visible, simply enter the data validation:

1.       Choose Method validation / calibration.

2.       Theme Name – Output theme name of the model run.

3.       Initial Attribute – Name of the initial real attribute, considered the first year of simulation (eg. Deforestation).

4.       % Allowed Error per Cell – Error tolerance level to be considered per cell (eg. 5% -. 0 to 5% error is considered a correct cell).

5.       Attribute for Validation – Attribute name to be validated, generated by the execution of the model. Attribute selected in Parameters to Save (eg. Desmatamento_out).

6.       Real Final Attribute – The actual final attribute name, year considered the validation.

7.       Number of Windows – Number of windows considered in the validation, 1 a comparison cell to cell, numbers greater than 1 is a cluster of cells (eg. 10, window comparisons will be made up for 1 to 10 windows).

8.       Save into Databse – Select whether to save the map of the differences in each of the windows in the database (Note about the values: Zero - Correct; Negatives - Error by Committee; Positives - Error for failure to act).
