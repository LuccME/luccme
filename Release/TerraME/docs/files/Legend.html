Legend

Type that defines how to color an attribute to be observed. It is used only with a map Observer. The configuration of a legend can be changed visually by double clicking the graphical interface along the simulation.

Parameters

  • colorBar: A table where each position is a table with a 'color', a 'value', and a 'label' (optional). Colors can be described as string ("red", "green", "blue", "white", "black", "yellow", "brown", "cyan", "gray", "magenta", "orange", "purple", and their light and dark compositions, such as "lightGray" and "darkGray"), or as tables with three integer numbers representing RGB compositions. Labels are strings that will be shown in the graphical interface describing the colors.
  • font: A string with the font used to draw each Agent. Default is "Symbol".
  • fontSize: An integer positive number indicating the font size. Default is 12.
  • grouping: A string with the strategy to slice and color the data. See below.
    • Grouping Description Compulsory parameters Optional parameters
    "equalsteps" The values are divided into a set of slices with the same range. Each slice is associated to a given color. Equalsteps require only two colors in the colorBar, one for the minimum and the other for the maximum value. The other colors are computed from a linear interpolation of the two colors. colorBar, maximum, minimum, slices font, fontSize, precision, symbol, type, width
    "quantil" Aggregate the values into slices with approximately the same size. Values are ordered from lower to higher and then sliced. This strategy uses two colors in the same way of equalsteps. colorBar, maximum, minimum, slices precision, type, width
    "stdeviation" Define slices according to the distribution of a given attribute. Values with similar positive or negative distances to the average will belong to the same slice. colorBar, stdColorBar precision, stdDeviation, type, width
    "uniquevalue" Map each attribute value to a given color. Attributes with type string can only be sliced with this strategy. colorBar type, width
  • maximum: The maximum value of the attribute (used only for numbers).
  • minimum: The minimum value of the attribute (used only for numbers).
  • precision: The number of decimal digits for slicing. It must be an integer number greater than zero. It indicates that differences less than 10^(-digits) will not be considered. It means that, for instance, if a slice is in the interval [1.0,2.0] and precision is 2 (0.01), a value 0.99 might belong to such slice.
  • slices: The number of colors to be used for plotting. It must be an integer number greater than one.
  • stdColorBar: A table just as colorBar. It is needed only when standard deviation is the chosen strategy.
  • stdDeviation: When the grouping mode is stddeviation, it has to be one of "full", "half" "quarter", or "none".
  • symbol: A symbol to draw Agents. It can be one of: "none", "ellipse", "rect", "diamond", "triangle", "dtriangle", "utriangle", "ltriangle", "rtriangle", "cross", "xcross", "hline", "vline", "asterisk", "star", "hexagon". Default is an arrow.
  • type: The type of the attribute to be observed. It has to be one of "bool", "number", "string", and "datetime" (an ordered string).
  • width: The width of the line to be drawn. Used for drawing Neighborhoods (default is 10).

Usage

coverLeg = Legend {
    grouping = "uniquevalue",
    colorBar = {
        {value = 0, color = "white"},
        {value = 1, color = "red"},
        {value = 2, color = "green"}
    }		
}

deforLeg = Legend {
    grouping = "equalsteps",
    slices = 10,
    colorBar = {
        {value = 0, color = "green"},
        {value = 1, color = "red"}
    }
}
